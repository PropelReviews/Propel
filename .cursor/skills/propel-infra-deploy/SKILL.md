# Propel Infra & Deploy

AWS `us-east-1`, two accounts: **beta** (536270449640, `beta.propel.ninja`) and **prod** (616938645090, `propel.ninja`). API + Dagster on ECS Fargate behind ALB; app SPA and landing on S3 + CloudFront; Aurora Serverless v2; secrets in Secrets Manager; CI auth via GitHub OIDC → `PropelTerraform` role.

## Local stack (docker compose, project `propel`)

`postgres` :5432 · `backend` :8000 · `frontend` :5173 · `ingestion` (Dagster) :3001 · `frontend-landing` :5174 · `frontend-landing-static` :8080 (profile `landing`). The devcontainer auto-starts only postgres/backend/frontend (`runServices`); start `ingestion` manually. Env from root `.env` (+ optional `.env.ingestion.local`).

## Terraform

```
infrastructure/terraform/
├── modules/        # network, database, api (+ ingestion.tf), frontend, landing, dns, stack
├── environments/   # beta/, prod/  (state: per-account S3 bucket + propel-tf-locks DynamoDB)
└── bootstrap/      # IAM trust/permission JSON (manual, not TF-managed)
```

```bash
aws sso login --sso-session propel
export AWS_PROFILE=propel-beta            # or propel-prod
cd infrastructure/terraform/environments/beta
terraform init && terraform plan
terraform fmt -recursive                  # CI enforces fmt -check; TF version 1.9.8
```

Apply beta before prod (prod reads beta's NS records for DNS delegation). `app.auto.tfvars.json` is generated by `scripts/generate-app-tfvars.sh` from GitHub Actions vars/secrets — app config is never hardcoded in TF.

## Scripts (`scripts/`)

| Script | Purpose |
|---|---|
| `setup.sh` / `dev.sh` | devcontainer postCreate (deps, AWS SSO config) / postAttach health checks |
| `dev-ingestion-secrets.sh pull` | GitHub App dev creds from Secrets Manager → `.env.ingestion.local` |
| `deploy-api.sh <beta\|prod>` | build prod image, push ECR, force ECS redeploy (API + ingestion) |
| `deploy-frontend.sh` / `deploy-landing.sh <env>` | vite build → s3 sync → CloudFront invalidation |
| `restart-api.sh <env>` | ECS redeploy without rebuild |
| `generate-app-tfvars.sh` | CI: GitHub vars/secrets → tfvars json |

Deploy scripts need `AWS_PROFILE=propel-beta|propel-prod` locally (CI uses OIDC).

## CI/CD (`.github/workflows/`)

- `ci.yml` (PRs + main): `terraform` (fmt + validate both envs), `backend` (ruff + pytest w/ Postgres service), `ingestion-integration` (alembic + target-propel tests), `frontend` (eslint, prettier, tsc, vitest w/ Playwright Chromium, both builds).
- `deploy-beta.yml`: push to `main` → terraform apply + deploy api/frontend/landing to beta.
- `deploy-prod.yml`: `v*` tag or successful beta on main → same flow with `prod` environment approval gate.

Docs: `docs/deployment/bootstrap.md` (one-time setup), `docs/deployment/cicd.md`, `docs/deployment/aws-sso.md`.

## Gotchas

- `SESSION_SECRET` is generated by Terraform (not in GitHub secrets).
- `ZITADEL_CLIENT_SECRET` and `ZITADEL_MGMT_TOKEN` are GitHub Environment secrets → `app_secrets` via `generate-app-tfvars.sh`.
- State backends are hardcoded in each env's `backend.tf` — never change bucket/lock names.
- Migrations run automatically when the API container starts; the ingestion ECS service sets `SKIP_MIGRATIONS=1`. Use expand/contract migrations for rolling deploys.
- Smoke check after deploy: `curl https://api.beta.propel.ninja/health`.
- No CloudWatch app logs — observability goes to PostHog via OTEL.
