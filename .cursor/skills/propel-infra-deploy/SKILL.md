# Propel Infra & Deploy

AWS `us-east-1`, **prod** account (616938645090, `propel.ninja`). Beta was decommissioned (June 2026). API + Dagster on ECS Fargate behind ALB; app SPA and landing on S3 + CloudFront; Aurora Serverless v2; secrets in Secrets Manager; CI auth via GitHub OIDC тЖТ `PropelTerraform` role.

## Local stack (docker compose, project `propel`)

`postgres` :5432 ┬╖ `backend` :8000 ┬╖ `frontend` :5173 ┬╖ `ingestion` (Dagster) :3001 ┬╖ `frontend-landing` :5174 ┬╖ `frontend-landing-static` :8080 (profile `landing`). The devcontainer auto-starts only postgres/backend/frontend (`runServices`); start `ingestion` manually. Env from root `.env` (+ optional `.env.ingestion.local`).

## Terraform

```
infrastructure/terraform/
тФЬтФАтФА modules/        # network, database, api (+ ingestion.tf), frontend, landing, dns, stack
тФЬтФАтФА environments/   # prod/ (active); beta/ retained for reference/rebuild
тФФтФАтФА bootstrap/      # IAM trust/permission JSON (manual, not TF-managed)
```

```bash
aws sso login --sso-session propel
export AWS_PROFILE=propel-prod
cd infrastructure/terraform/environments/prod
terraform init && terraform plan
terraform fmt -recursive                  # CI enforces fmt -check; TF version 1.9.8
```

`app.auto.tfvars.json` is generated by `scripts/generate-app-tfvars.sh` from GitHub Actions vars/secrets тАФ app config is never hardcoded in TF.

## Scripts (`scripts/`)

| Script | Purpose |
|---|---|
| `setup.sh` / `dev.sh` | devcontainer postCreate (deps, AWS SSO config) / postAttach health checks |
| `dev-ingestion-secrets.sh pull` | GitHub App dev creds from Secrets Manager тЖТ `.env.ingestion.local` |
| `deploy-api.sh <beta\|prod>` | build prod image, push ECR, force ECS redeploy (API + ingestion) |
| `deploy-frontend.sh` / `deploy-landing.sh <env>` | vite build тЖТ s3 sync тЖТ CloudFront invalidation |
| `restart-api.sh <env>` | ECS redeploy without rebuild |
| `generate-app-tfvars.sh` | CI: GitHub vars/secrets тЖТ tfvars json |

Deploy scripts need `AWS_PROFILE=propel-prod` locally (CI uses OIDC).

## CI/CD (`.github/workflows/`)

- `ci.yml` (PRs + main): `terraform` (fmt + validate prod), `backend` (ruff + pytest w/ Postgres service), `ingestion-integration` (alembic + target-propel tests), `frontend` (eslint, prettier, tsc, vitest w/ Playwright Chromium, both builds).
- `deploy-prod.yml`: `v*` tag or manual `workflow_dispatch` тЖТ terraform apply + deploy api/frontend/landing with `prod` environment approval gate.

Docs: `docs/deployment/bootstrap.md` (one-time setup), `docs/deployment/cicd.md`, `docs/deployment/aws-sso.md`.

## Gotchas

- `JWT_SECRET` is generated by Terraform (not in GitHub secrets).
- State backends are hardcoded in each env's `backend.tf` тАФ never change bucket/lock names.
- Migrations run automatically when the API container starts; the ingestion ECS service sets `SKIP_MIGRATIONS=1`. Use expand/contract migrations for rolling deploys.
- Smoke check after deploy: `curl https://api.propel.ninja/health`.
- No CloudWatch app logs тАФ observability goes to PostHog via OTEL.
